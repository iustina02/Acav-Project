1. Abstract

2. Introduction

Aceasta este o aplicaţie bazata pe (micro-)servicii Web oferind modalităţi de vizualizare avansată a conţinuturilor muzicale pe care le asculta/partajeaza online anumite (grupuri de) persoane -- se vor prelua informaţii de la servicii (streaming) de profil. Ca exemple de vizualizări pot fi considerate: figurarea grafului melodiilor favorite ale mai multor utilizatori având o anumita localizare geografică ori preocupări comune, relaţiile între melodii similare pe baza meta-datelor asociate fiecarei productii muzicale, generarea unor grupuri de utilizatori împărtăşind interese comune (e.g., pasionatii de muzica rock progresiv compusa intre anii 1970 si 1977 si interpretata de grupuri europene), eventual folosindu-se termeni de conţinut (tag-uri) asociate melodiilor. Vizualizarile create vor putea fi exportate ca fisiere PNG si SVG.

3. Structure

Etape

  • 1. FrontEnd Desing

    • Pentru front desing vom folosi HTML + CSS s vom incerca ca interfata pentru utilizator sa fie cat mai simpla. Aceasta schimbandu-se in functie de actunile utilizatorului.

  • 2. BackEnd

    • Pentru partea de server vom folosi ECMAScript. Astfel vom putea crea o aplicatie Single-Page ce va fi usor de chestionat.

  • 3. DataBase

    • Datele pentru crearea unui cont de catre utilizator vor fi stocate intr-o baza de date ce va fi interogata pentru eventuale logari pe site sau va fi modificata in cazul in care utilizatorul doreste sa-si modifice anumite date.

  • 4. API

    • Vom folosi urmatoarele API-uri pentru ca site-ul nostru ca sa fie cat mai complet si fluent in transmiterea si distribuirea datelor cu continut muzical si nu numai:
    • Openwhyd API

    • E-mail System

    • API pentru download-ul de grafuri in formatul cerut



4. Interactiune cu utilizatorul

Actiunile posibile pentru utilizatorul neconectat:

  • SignUp

    • Crearea unui cont introducand urmatoarele date:
      Nume
      Prenume
      Email
      Parola
      Optionale: numar telefon, data nastere, genuri muzicale, oras, tara


  • LogIn

    • Logarea intr-un cont existent introducant datele:
      Email
      Parola

  • Reset Password

    • Resetarea parolei introducand emailul:
      Email

    Astfel utilizatorul va avea posibilitatea de a-si schimba parola in cazul in care aceasta o uita.


    Actiunile posibile pentru utilizatorul conectat:

  • Vizualizare Profil

    • Redirectionarea cu ajutorul unui buton catre pagine de profil a utilizatorului unde acesta poate vedea urmatoarele date:
      Nume
      Prenume
      Email
      Numar telefon
      Data nastere
      Genuri muzicale
      Oras
      Tara


  • ShareMusic

    • Utilizatorul poate sa distribuie melodiile preferate ce vor aparea in "news".

  • Vizualizare Continut Muzical

    • Va avea oportunitatea de a asculta continutul muzical pe care il alege si totodata de a vedea anumite "grafuri" in legatura cu muzica apreciata de anumite categorii de persoane (ex: pozitia geografica, ocupatie, varsta).

  • Primirea de sugestii

    • Va avea oportunitatea de a primi anumite sugesti legate de melodii ce i-ar putea placea, avand aceleasi taguri cu cele deja apreciate sau distribuite.

  • Modificarea datelor personale

    • Acesta isi poate modifica datele introduse in procesul de inregistrare si poate adauga campurile optionale (daca nu au fost completate)

  • Log out / Delete Account

    • Acesta are intotdeauna posibilitatea de a se deconecta de la cont sau de a sterge complet contul si totodata datele legate de activitatea acestuia.

  • Openwhyd API
    • Users can access and download data that they contributed to Openwhyd

    Users can even use their Openwhyd account from 3rd-party software. The list of tracks can be downloaded in several formats: JSON or as a list of URLs.

    GET Parameters
    format: “json” or “links”
    limit: number of posts to return (optional, default=20)
    after: identifier of the post (a.k.a. pId) from which entries must be returned (optional, for pagination)

    Acest API poate fi folosit pentru mai multe actiuni, verificati sursa:
    Openwhyd

  • EmailJS API
    • Sends an email based on the specified email template and the passed dynamic parameters. The email is sent via the specified email service or via the default service if "default_service" is passed.

    Response formats is JSON or Text Sursa: EmailJS API
